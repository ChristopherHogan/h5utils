h5utils

This package contains a few utilities that I wrote for accessing data in the HDF5 format developed by NCSA at the University of Illinois. You will need the HDF5 library, available from the HDF home page. The author of this package is Steven G. Johnson (stevenj@alum.mit.edu).

To compile and install the package, just type the usual: 

./configure
make
make install

You can use make uninstall to get rid of all the installed files.

Note that you need to have certain prerequisites, like the PNG and zlib libraries and/or GNU Octave, in order to compile all of the utilities, as described below. If you don't have these, then only some of the utilities will be compiled.

The utilities are:

h5topng

This is a program to extract a two-dimensional slice from an HDF5 dataset and export it as an image in PNG format. It is designed for quick-and-dirty examination of data, and for batch-processing of data on the command-line. However, it does support niceties like scaling, skewing, superimposing contours from another file, and a couple of different color maps. PNG is a free image format, and is a recommendation of the World Wide Web Consortium. You will need to get the free libpng library if you don't have it already. libpng and lots of other stuff can be found at the PNG home page.

You will also need zlib (libz) if you don't already have it, for image compression (see the zlib Home Page).

The h5topng source code includes the files writepng.c and writepng.h, which provide self-contained subroutines for writing out a PNG image from a two-dimensional array of floating-point values. You might find this useful if you want to directly output images from your own scientific programs.

h5topng includes a man page; you can also run h5topng -h for a terse listing of options.

h5totxt

This program outputs 2d slices of HDF5 files as ASCII text in comma-delimited format, suitable for importing into a spreadsheet. (You can use a different delimiter with the -s option.) Run h5totxt -h for a terse listing of its options; there is also a man page.

h5fromtxt

In some sense the inverse of h5totxt, this program takes text input and outputs a multi-dimensional numeric dataset in an HDF5 file. The input numbers can be delimited by arbitrary characters, and it can automatically guess the dimensions of one- and two-dimensional input. Multi-dimensional datasets can also be written by specifying their dimensions explicitly. Run h5fromtxt -h for a terse listing of its options; there is also a man page.

h5fromh4

The h5fromh4 program takes a file in the old HDF4 format and converts the dataset contained therein to an HDF5 file. (Currently, only a single dataset per HDF4 file is supported. The HDF5 developers are supposedly working on a more general conversion utility.)

The HDF4 libraries need to be installed on your system for this program to be compiled and installed (otherwise it will be skipped).

h5tov5d

h5tov5d is a program to convert multidimensional HDF5 datasets to "v5d" files, the format used by Vis5d. Vis5d is a free volumetric visualization program for looking at 3, 4, or even 5-dimensional datasets. It allows you to view slices, isosurfaces, volume renderings, contours, and much more. (4 and 5 dimensional datasets are rendered via time-series animation and multiple "variables.") Being based on OpenGL, Vis5d also lets you rotate and manipulate its graphs in real time.

In order to compile h5tov5d, you need to first download and compile Vis5d (available from the Vis5d Home Page). It is not sufficient to download the precompiled binaries, as they don't include the necessary object files for writing the v5d data format. Then, when configuring h5utils, you should use: 

./configure --with-v5d=dir

Here, dir should be the path of the Vis5d source directory which you downloaded and compiled. Once you have successfully compiled h5tov5d, you may delete the Vis5d source directory if you wish.

Alternatively, you can download my Vis5D 5.2a package, which installs the relevant Vis5d routines as shared libraries. In this case, you won't need to specify --with-v5d.

See the h5tov5d man page for usage instructions, or run h5tov5d -h for a terse listing of options.

h5read.oct

This is an Octave plug-in for importing 2d slices from HDF5 datasets into Octave, where you can then manipulate and plot the data. GNU Octave is a free Matlab-like data analysis and numerical computing tool; you can download it or find more about it at the Octave home page.

You will need to have Octave already installed in order to compile this. Once it is compiled, make install places h5read.oct in the global site directory for Octave plugins (OCTAVE_HOME/libexec/octave/site/oct/ARCH/).

In Octave, type help h5read for instructions.

arrayh5

The files arrayh5.c and arrayh5.h define some data structures and utilities to let you grab multi-dimensional datasets (or slices thereof) from HDF5 files using a simplified interface. You might find it useful in your own programs, or just to help you learn how to use HDF5.
